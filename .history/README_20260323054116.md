## Local Development

### 1. Clone the repository

```bash
git clone https://github.com/Weerapats/secure-note-app.git
cd secure-note-app
```

### 2. Configure environment variables

```bash
cd backend
cp .env.example .env
```

Open `.env` and fill in your values:

```dotenv
PORT=3001
SECRET_TOKEN=YOUR_SECRET_TOKEN
POCKETHOST_TOKEN=your-pockethost-token
```

> вљ пёЏ **Never commit `.env` to Git.** It is already listed in `.gitignore`.

### 3. Start the backend

```bash
cd backend
npm install
npm start
# Server running at http://localhost:3001
```

### 4. Start the frontend

Open a new terminal:

```bash
cd frontend
npm install
npm run dev
# App running at http://localhost:5173
```

Open `http://localhost:5173` in your browser and sign in using the `SECRET_TOKEN` from `backend/.env`.

## API Reference

All requests go to `http://localhost:3001` locally or `https://secure-note-app-lz21.vercel.app` in production.

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `GET` | `/api/verify` | вњ… Yes | Validate the token |
| `GET` | `/api/notes` | вќЊ No | Get all notes |
| `POST` | `/api/notes` | вњ… Yes | Create a new note |
| `DELETE` | `/api/notes/:id` | вњ… Yes | Delete a note by ID |

### Authorization header

All protected endpoints require:
```
Authorization: <your SECRET_TOKEN>
```

### POST `/api/notes` вЂ” Request body

```json
{
  "title": "My Note",
  "content": "Note content here."
}
```

### HTTP Status Codes

| Code | Meaning | When |
|------|---------|------|
| `200` | OK | Successful GET or DELETE |
| `201` | Created | Note successfully created |
| `400` | Bad Request | Missing title or content |
| `401` | Unauthorized | Wrong or missing token |
| `404` | Not Found | Note ID does not exist |
| `500` | Server Error | Unexpected server exception |



## Deployment on Vercel

This project is deployed as **two separate Vercel projects**.

### Deploy Backend

1. Go to [vercel.com](https://vercel.com) в†’ **Add New Project**
2. Import this repo в†’ set **Root Directory** to `backend`
3. Framework Preset: **Other**
4. Add Environment Variables:
   - `SECRET_TOKEN` вЂ” your secret token
   - `POCKETHOST_TOKEN` вЂ” your PocketHost token
5. Click **Deploy**

### Deploy Frontend

1. **Add New Project** again в†’ import the same repo
2. Set **Root Directory** to `frontend`
3. Framework Preset: **Vite**
4. Add Environment Variable:
   - `VITE_API_URL` вЂ” your backend Vercel URL + `/api` (e.g. `https://your-backend.vercel.app/api`)
5. Click **Deploy**

> вљ пёЏ After changing environment variables, always **Redeploy** the project for changes to take effect.

## Security

- `.env` is git-ignored вЂ” secrets never enter version control
- `SECRET_TOKEN` is never written in any frontend source file
- `POCKETHOST_TOKEN` only ever exists in `process.env` on the server
- Token is validated server-side on every write request via `requireAuth` middleware
- All production traffic is encrypted with **HTTPS / TLS** via Vercel
- Token is stored in `sessionStorage` вЂ” automatically cleared when the browser closes
